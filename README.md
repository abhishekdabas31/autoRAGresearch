# autoRAGresearch 🔬

> Autonomous RAG pipeline optimizer. The agent iterates. You sleep.
> Inspired by Andrej Karpathy's [autoresearch](https://github.com/karpathy/autoresearch).

## The Idea

autoRAGresearch gives an AI agent a real RAG pipeline and lets it experiment autonomously overnight. The agent reads the current `rag_pipeline.py`, forms a hypothesis for improvement, modifies it (chunk strategy, embedding model, reranker, prompt, top-k, etc.), runs a fixed-time evaluation against a static query set, checks if the score improved, commits the change if it did, reverts if not, and loops.

The human never touches the Python. They only write `program.md` — a Markdown file that gives the agent research direction.

The output is a **git history of autonomous improvements**, a live leaderboard, and a best-config snapshot that outperforms the naive human-tuned baseline.

## Quick Start

```bash
# 1. Clone and install
git clone https://github.com/abhishekdabas31/autoRAGresearch.git
cd autoRAGresearch
pip install -r requirements.txt

# 2. Generate the default corpus and eval set
python setup_data.py

# 3. Pull local models (one-time)
ollama pull llama3.2:3b
ollama pull nomic-embed-text

# 4. Set your Anthropic API key (for the loop agent only)
cp .env.example .env
# Edit .env with your key

# 5. Initialize the git branch for experiments
git checkout -b autorag-experiments

# 6. Run the baseline eval to see your starting score
python eval.py

# 7. Edit program.md to set your research direction (or use the default)

# 8. Start the loop. Go to sleep.
python loop.py
```

## How It Works

```
LOOP (runs indefinitely until Ctrl+C):
  1. READ    → load current rag_pipeline.py + program.md + recent history
  2. PROMPT  → ask Claude: "propose ONE targeted modification to improve the score"
  3. WRITE   → save the agent's proposed rag_pipeline.py (validate syntax first)
  4. EVAL    → run eval.py with timeout
  5. JUDGE   → compare new score vs best — commit if improved, revert if not
  6. LOG     → append experiment record to experiments.jsonl
  7. UPDATE  → rewrite README.md leaderboard
  8. GOTO 1
```

## The Three Files That Matter

| File | Who edits it | What it is |
|---|---|---|
| `rag_pipeline.py` | The AI agent | The research canvas |
| `program.md` | The human | The research direction |
| `eval.py` + `corpus_prep.py` | Nobody (fixed) | The arena |

## What Makes This Different From Optuna

- **Optuna** searches a predefined parameter space you specify in advance.
- **autoRAGresearch** lets the agent *define the search space dynamically* — it can invent new chunking strategies, rewrite retrieval logic, change prompt structure, add techniques from its training knowledge.
- The agent can propose changes no parameter grid would have included.
- Over hundreds of iterations, it discovers non-obvious interaction effects.
- The git history is human-readable — you see *why* each change worked.

This is the autoresearch thesis applied to RAG: **the bottleneck isn't compute, it's the search space. Let the agent define the search space.**

## CLI Options

```bash
python loop.py                          # run indefinitely
python loop.py --experiments 50         # run exactly 50 experiments
python loop.py --model claude-opus-4-6  # use a different agent model
python loop.py --dry-run                # propose changes without committing
python loop.py --sleep 10              # pause 10s between experiments
```

## Expected Improvement Arc

| Phase | What gets discovered | Typical gain |
|---|---|---|
| Early (1-20) | Better chunk size, overlap tuning | +0.02–0.05 |
| Mid (20-60) | Hybrid retrieval, prompt improvements | +0.03–0.07 |
| Late (60-120) | Reranking, query expansion, HyDE | +0.02–0.04 |
| Expert (120+) | Combination effects, edge cases | +0.01–0.02 |

## Dependencies

The RAG pipeline runs **entirely on local models** via Ollama. Claude is only used in `loop.py` as the optimization agent. Zero closed-model dependency in the pipeline itself.
